# Excalidraw Diagram Skill — Design

**Date:** 2026-04-03
**Status:** Approved
**Owner:** Greg

## Problem

Diagrams generated by LLMs writing raw Excalidraw JSON have layout issues: arrows pointing wrong directions, overlapping text, poor spacing. The proven approach (Python generator scripts like `build-karpathy-loop.py`) produces excellent diagrams because layout is computed mathematically, not guessed. But there's no skill that encodes this method — every new diagram requires rediscovering the pattern.

Additionally, there's no review loop. The pipeline docs say "Claude reads the PNG, confirms quality or adjusts" but this step has no automation, no checklist, and no structured feedback.

## Decisions

- **Rouge owns the tooling** — renderer, design rules, and skill all live in The Rouge. AI-Factory keeps its copies for Substack; they diverge naturally.
- **Rouge invokes independently** — no AI-Factory dependency for diagram generation.
- **One skill, two phases** — Phase 1 generates via Python script, Phase 2 reviews rendered PNG and iterates.

### Skill: `excalidraw-diagram`

Single skill with two phases. Invoked when any Claude session needs to create or update an Excalidraw diagram.

#### Phase 1: Generate

The skill instructs Claude to:

1. **Read reference materials:**
   - `docs/design/diagram-design-rules.md` — the rulebook
   - `docs/diagrams/build-karpathy-loop.py` — the canonical generator pattern

2. **Gather the diagram brief** from the user or context:
   - What the diagram shows (flow, architecture, state machine, etc.)
   - Nodes and their roles (process, decision, success, warning)
   - Connections and flow direction
   - Any specific layout constraints

3. **Write a Python generator script** that:
   - Uses 8px grid alignment for all dimensions
   - Computes coordinates mathematically (no guessing)
   - Follows the `build-karpathy-loop.py` pattern: `make_rect()`, `make_diamond()`, `make_arrow()` helpers
   - Applies the colour palette with semantic roles (process=blue, decision=coral, success=green, warning=amber)
   - Sizes shapes per the rulebook (Hero 304x152, Primary 216x88, Standard 120x80, Diamond 1.4x)
   - Computes text padding (H: 20% of width, V: 15% of height)
   - Routes arrows: prefer straight H/V, feedback loops exit right with 80-96px offset, max 3 bends
   - Keeps text fill <55% of shape interior
   - Uses font sizes at 20-25% of box height
   - Sets roughness 1.0, fillStyle hachure, strokeWidth 2.5
   - Saves to `docs/diagrams/<name>.excalidraw`

4. **Run the generator:**
   ```bash
   python3 docs/diagrams/build-<name>.py
   ```

5. **Render to PNG:**
   ```bash
   cd tools/diagrams && uv sync && uv run python render_excalidraw.py <path>.excalidraw --output <path>.png --scale 2
   ```

#### Phase 2: Review + Iterate

After rendering, the skill instructs Claude to:

1. **Read the rendered PNG** using the Read tool (multimodal image reading)

2. **Check against design rules checklist:**

   **Layout:**
   - [ ] Nodes aligned to 8px grid
   - [ ] 80-88px spacing between adjacent ranks
   - [ ] No overlapping elements
   - [ ] Generous whitespace (60-30-10 rule)
   - [ ] Max 10 boxes (split into multiple diagrams if larger)

   **Arrows:**
   - [ ] All arrows point in the correct direction
   - [ ] No arrows crossing (or crossing at 90 degrees if unavoidable)
   - [ ] Max 3 bends per arrow
   - [ ] Feedback loops route outside, not through other nodes
   - [ ] Arrow labels are legible and positioned clearly

   **Text:**
   - [ ] Text fill <55% of shape interior
   - [ ] Minimum 12px font size everywhere
   - [ ] Diamond labels are 2-4 words max
   - [ ] No text overflows its container
   - [ ] Labels are short (2-4 words per box)

   **Colour:**
   - [ ] Max 4-5 semantic colours
   - [ ] Stroke darker than fill
   - [ ] Colour encodes meaning (not decoration)

   **Overall:**
   - [ ] Diagram is immediately understandable without reading a legend
   - [ ] Flow direction is clear (top-to-bottom or left-to-right)
   - [ ] The diagram tells the story it's supposed to tell

3. **If issues found:**
   - Modify the Python generator script to fix the specific issues
   - Re-run the generator
   - Re-render to PNG
   - Re-review
   - **Max 3 iterations.** After 3, flag remaining issues for human review.

4. **If all checks pass:**
   - Commit the generator script, .excalidraw, and .png
   - Report completion

### What the Skill Does NOT Do

- **Does not write raw Excalidraw JSON.** Always generates via Python script. This is the core design decision — mathematical layout beats LLM coordinate guessing.
- **Does not use Mermaid conversion.** The Python generator approach produces better results with more control.
- **Does not handle D2 diagrams.** Separate tooling.
- **Does not generate diagrams for articles.** AI-Factory's `write-article` skill has its own diagram pipeline via `stacks/substack/`. This skill is for architectural/documentation diagrams.

## First Use: Regenerate V2 Process Map

After the skill is created, use it to regenerate `docs/diagrams/v2-process-map` properly:

1. Invoke the skill with the V2 process map brief (from `docs/design/v2-process-map.md`)
2. It writes `docs/diagrams/build-v2-process-map.py` following the `build-karpathy-loop.py` pattern
3. Generates `.excalidraw`, renders `.png`
4. Reviews and iterates
5. Replaces the current LLM-generated V2 diagram

This proves the skill works end-to-end.
